Git:

O Git é um projeto de código aberto maduro e com manutenção ativa desenvolvido em 2005 por Linus Torvalds, o famoso criador do kernel do sistema operacional Linux.

É um Software de controle de versão (VCS) e dentre as suas vantegens estão:

  • Controle de Histórico
  • Trabalho em Equipe
  • Ramificação do Projeto
  • Segurança
  • Organização

    • GitHub:

    Já o GitHub, é um Repositório Remoto, mais conhecido como Rede social para Programadores, deu início em 2008, quando foi desenvolvido por Chris Wanstrath, J. Hyett, Tom Preston-Werner e Scott Chacon usando Ruby on Rails. Em 10 anos o Github teve mais de 3 Milhões de usuários e mais de 5 Milhões de repositórios salvos, quando em 2018 foi vendida pela grande Microsoft. E dentre suas principais vantagens podemos destacar:

  • Repositórios Ilimitados
  • Hospedagem de Código-fonte
  • Características de rede social
  • Colaboração
  • Forks/Biforcação de Códigos

    • Principais Comandos:

  • git init - Inicializa um repositório Git na pasta em questão
  • git clone - Faz um clone do repositório remoto em seu repositório local
  • git commits - adiciona as alterações para o histórico do repositório e atribui um nome ao commit
  • git push - mais usado para publicar modificações locais a um repositório central.

    • Podemos verificar que Git E Github são coisas bem diferentes uma da outra mas que juntos desempenham um papel importantíssimo para a programação, e hoje são um dos Softwares mais ultilizados em todo mundo na área da tecnologia.
